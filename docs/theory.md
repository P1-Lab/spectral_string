# Spectral String — Theory

Spectral String is based on the premise that **sound does not need to be generated from discrete events**, but can instead be **resolved from continuous mechanical excitation into stable spectral fields**.

## Core Concept

A traditional instrument:
- excitation → resonance → decay → silence

Spectral String:
- excitation → continuous state mapping → spectral stabilization → persistent harmonic field

The system treats the string not as a note generator, but as a **boundary condition for a dynamic spectral solver**.

## Non-Random Excitation (NRE)

NRE is the governing model of the system:

> Physical input is never interpreted as gesture, only as energy topology.

This avoids:
- velocity layers
- MIDI quantization
- event triggering

Instead:
- energy density
- spectral centroid drift
- harmonic stability fields

## String as Field Generator

The string behaves as:
- a 1D mechanical oscillator
- a boundary condition for harmonic collapse
- a controllable instability surface

When excited, it does not “play a note” — it defines a **temporary harmonic geometry**.

## Determinism Principle

All transformations are deterministic:
- no stochastic synthesis
- no randomized timbre modulation
- no sample playback variance

Every instantaneous system state maps to a reproducible spectral output.
