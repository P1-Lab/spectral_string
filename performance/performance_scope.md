# Performance Scope — Spectral String System

## Version
v1.0 — Canonical Physical Instrument Definition

---

## 1. Instrument Identity

The current Spectral String system is a **physical acoustic instrument with spectral extension**, not a digital controller.

It is defined by:

- continuous string excitation
- mechanically stable lever-based tuning system
- deterministic spectral resolution layer (NRE)
- real-time coupling between physical vibration and modeled harmonic field

This system is **complete as an acoustic instrument without electronics**.
The electronic layer extends harmonic interpretation but does not define sound production.

---

## 2. What This Instrument IS Built For

### 2.1 Primary Performance Use

This iteration is explicitly optimized for:

- solo performance
- duet / consort playing
- small ensemble integration
- sustained harmonic exploration
- gesture-based continuous pitch control

It is a **repertoire instrument**, not a trigger interface.

---

### 2.2 Core Musical Capabilities

The system is designed for:

- continuous pitch morphing (fretless behavior)
- microtonal stability across long phrases
- harmonic field interaction (dual-string interference patterns in Duo models)
- dynamic bow/pluck/pressure articulation (depending on performer technique)
- stable re-tuning between A=440 and A=415 reference systems

---

### 2.3 Ensemble Role

In ensemble contexts, the instrument functions as:

- harmonic bridge layer (between acoustic and electronic instruments)
- sustained tonal anchor
- spectral texture generator
- non-tempered pitch surface

It is NOT designed as:

- rhythmic percussion replacement
- MIDI-style note generator
- discrete event sequencer

---

## 3. Physical Interaction Model

Performance is governed by:

### 3.1 Gesture Input Types

- pluck (impulse excitation)
- slide (continuous excitation)
- pressure modulation (microtonal shaping)
- bowing (optional extension mode)
- lever-based global tuning shift

---

### 3.2 Response Behavior

The system produces:

- continuous spectral unfolding
- non-linear harmonic bloom
- stable decay fields (no artificial gating)
- physically anchored pitch drift resistance

---

## 4. What This Version DOES NOT Include

This is critical for architectural integrity.

Current system explicitly excludes:

- arco-specific optimization layer (reserved for future expansion)
- polyphonic multi-string orchestration beyond Mono/Duo models
- digital sequencing or DAW-style event generation
- preset-based “sound design mode”
- MIDI control surface abstraction

---

## 5. Future Expansion Models (Non-Canonical Extensions)

These are planned *but not part of current build*:

### 5.1 Extended Performance Models
- arco-optimized string response system
- multi-bow interference modeling
- extended sustain engine (controlled feedback excitation)

### 5.2 Ensemble Scale Models
- triad / quad string architectures
- distributed spectral string consort systems
- spatialized instrument networks

### 5.3 Augmented Control Systems
- gesture capture augmentation (optical / inertial)
- predictive excitation modeling
- performance memory layers (non-real-time recall of gesture patterns)

---

## 6. Design Constraint Summary

Current instrument must remain:

- physically playable without electronics
- acoustically complete as a standalone instrument
- deterministic under identical mechanical excitation
- stable across tuning state transitions

Any future expansion must preserve these invariants.

---

## 7. Philosophical Performance Definition

The instrument is not a sound generator.

It is a **mechanical system for producing controlled spectral behavior through human gesture**.

The performer does not trigger sound.

The performer shapes a physical field.
